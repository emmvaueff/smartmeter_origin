# EMH Gateway

Custom Home Assistant Integration for EMH Gateway.

## Release-Checkliste

1. Aenderungen final pruefen und lokal testen.
2. Version in [custom_components/emh_gateway/manifest.json](custom_components/emh_gateway/manifest.json#L4) erhoehen.
3. Sicherstellen, dass [hacs.json](hacs.json) und Integration-Metadaten konsistent sind.
4. Aenderungen committen und nach GitHub pushen.
5. Git-Tag erstellen, z. B. `v1.0.1`, und pushen.
6. GitHub Release fuer das Tag erstellen (Release Notes hinzufuegen).
7. In HACS pruefen, ob das neue Release als Update erkannt wird.
